【阅读笔记】InstructERC：基于多任务检索增强大语言模型的对话情绪识别改进方法

Hao Teng
  2025-09-18 12:04:16 2025-09-18 12:04:16 Created   2025-09-22 16:08:30 2025-09-22 16:08:30 Updated  

简介

本文提出一种新方法 InstructERC，将 ERC（Emotion Recognition in Conversation）任务从判别式框架重构为基于大语言模型（LLMs）的生成式框架。
主要贡献：

  • 引入一个简单但有效的检索模板模块，帮助模型显式整合多粒度的对话监督信息；
  • 引入两项情感对齐的辅助任务——说话人识别与情感影响预测，用以隐式建模对话中的角色关系与未来情感倾向；
  • 首创性地通过“情感轮（feeling wheel）”在各基准间统一情感标签，以贴合真实应用场景。

问题定义

检索模板模块

通过微调 LLM ，ERC 任务被重构为 seq2seq 形式。作者构建了一个高效的检索模板模块，以弥合将 LLM 应用于具体 NLP 子任务时的gap。

对于 ERC任务，每个输入由四部分组成：

  • 指令（instructions）
    • 用于为模型提供明确定义的角色、ERC 任务的精确信息，以及输入对话文本的标准化格式,表示为
    • 相当于 prompt 的任务描述部分，告诉 LLM 扮演的角色与期望输出格式。
  • 历史内容（historical content）
    • 使用超参数“历史窗口 ”来指定纳入的历史轮次（含当前话语）及其说话者信息。目标话语 的
  • 标签说明（label statement）
    • 标签说明用于将模型输出限定在有限标签范围内，并让模型聚焦于当前被识别的话语
  • 示例检索（demonstration retrieval）
    • 从训练集构建领域库 ，移除说话者身份信息并平衡情感标签数量，保证示例不被说话者或标签分布偏差影响。
    • 对于待识别话语 ，从 中检索与之最相关的 ERC 示例。检索采用双向编码器 SBERT，为目标话语 和每个库中元素 生成 CLS 向量，并按余弦相似度排序，选择分数最高的示例 ：
    • $$d_{rvl_i}=\text{argmax}{d_j∈D{domain}}\text{SBERT}(u_i, d_j)$$

最终把四部分拼接成 LLM 的输入序列 xi，再交给 LLM 生成输出。
